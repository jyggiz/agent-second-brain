## Decision Filters

When evaluating ideas and tasks:

1. **Можно ли это делегировать?**  
   → Если да → задача бесполезна для меня, делегирую  
   → Если нет → переходим к следующему вопросу

2. **Нужно ли мне это? Приближает ли к целям?**  
   → Помогает ли это стать Lead Solution Engineer?  
   → Приближает ли это к финансовой независимости?  
   → Если нет обоим → отказ

3. **Отнимает ли это время от семьи?**  
   → Если да → обговорил ли с женой?  
   → Получил ли одобрение?  
   → Стоит ли оно того?

4. **Можно ли это автоматизировать или упростить с помощью AI?**  
   → Есть ли способ сделать это быстрее через Claude Code, Codex или другие инструменты?

```
Импликация: Сэкономим 10 часов в неделю команды = больше времени на разработку
Next Action: [ ] Внедрить Codex в CI/CD пайплайн до конца января
```

## Anti-Patterns (AVOID)

When processing entries DO NOT:

- ❌ Создавать абстрактные заметки без action items — каждая заметка должна иметь конкретное действие
- ❌ Делать задачи слишком размытыми типа "подумать о...", "изучить..." — всегда конкретные действия с дедлайнами
- ❌ Дублировать контент без синтеза — если информация повторяется, синтезировать в одну заметку с контекстом
- ❌ Создавать списки без приоритетов — всегда расставлять p1-p4 и указывать связь с целями
